# Unvibe overnight report — 2026-07-22

## Summary

Fixed the reported "the built app doesn't show in the Dock or menu bar" bug. Root
cause: the desktop app had **no committed icon assets at all** — the entire `app/build/`
directory was gitignored (root `.gitignore` ignored `build/`, and `app/.gitignore`
additionally ignored `build/icon.icns`). At runtime the menu-bar `Tray` was constructed
from an empty `NativeImage`, which macOS renders as an invisible menu-bar item, and the
packaged `.app` had no Dock icon. Two referenced packaging files
(`build/icon.icns`, `build/entitlements.mac.plist`) also did not exist, which would break
`npm run dist:dmg`.

This run generates a real, original Unvibe icon set, wires it through the build, hardens
the tray so it can never render invisibly again, adds the missing entitlements, and adds
regression tests so the assets can't silently disappear.

## Selected mission

Reliability / packaging (Run-1 rhythm): make the app actually appear in the Dock and menu
bar on macOS. Highest priority because an invisible app is a launch blocker regardless of
feature work.

## Problem found

- `app/src/main/main.ts` built the tray from `nativeImage.createFromPath(build/trayTemplate.png)`.
  That file (and `icon.png`, `icon.icns`) never existed on disk or in git, so the image was
  empty → **invisible menu-bar item**.
- `dist/assets/` was empty after `npm run build` because `scripts/build.mjs` silently
  swallowed the copy failure for the (nonexistent) icons.
- electron-builder config references `build/icon.icns` and `build/entitlements.mac.plist`;
  neither existed → generic Dock icon and a broken hardened-runtime packaging step.

## Work completed

- **New `app/scripts/gen-icons.py`** — dependency-free (stdlib `zlib` only) icon generator.
  Renders the Unvibe mark (deep near-black squircle + purple "U") from a signed-distance
  field so it stays crisp at every size. Emits `icon.png` (1024), `trayTemplate.png` (22),
  `trayTemplate@2x.png` (44), and a multi-resolution `icon.icns`.
- **Committed `app/build/` assets**: `icon.png`, `icon.icns`, `trayTemplate.png`,
  `trayTemplate@2x.png`, and a proper `entitlements.mac.plist` (JIT + network client for
  Electron under the hardened runtime).
- **`.gitignore` (root)** — added negations so `app/build/**` is tracked while other
  `build/` output stays ignored. **`app/.gitignore`** — dropped the `build/icon.icns` line.
- **`app/scripts/build.mjs`** — copies the retina tray asset too and now *warns loudly*
  instead of silently swallowing a missing-icon copy.
- **`app/src/main/main.ts`** — new `loadTrayImage()` with a disk → dock-icon → embedded
  base64 glyph fallback chain, and `setTemplateImage` handled centrally. The menu-bar item
  can no longer be invisible even if assets go missing.
- **New `app/test/icon-assets.test.ts`** — asserts the source assets exist and are
  well-formed (PNG dimensions, icns magic + `ic10` retina entry + self-consistent length,
  entitlements keys).
