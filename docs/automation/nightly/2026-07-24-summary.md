## Problems discovered

### Critical
1. **Sign-out is broken** — `web/app/api/v1/auth/signout/route.ts` calls
   `Store.revokeToken()` which is not in the interface or any implementation.
   Runtime throws `revokeToken is not a function`.
2. **History pagination broken in dev** — `web/app/api/v1/history/route.ts` calls
   `Store.historyPage()` which is not in the `Store` interface or `MemoryStore`.

### Type misalignment (54 TS errors)
3. `IncomingEvent` is missing `localDate`, `timezone` fields used in 4 files
4. `ProfileSummary` is missing `conceptsFamiliar`, `conceptsStrong` used in 2 files
5. `EventRecord` is missing `language` used in `skills.ts`
6. `plans.ts` is missing 11 exports imported by `memoryBillingStore.ts` and billing tests

### Test failures (17/36 failing in web/)
7. 4 security tests fail due to missing methods/constants in MemoryStore
8. 11 billing tests fail due to missing exports from `plans.ts`
9. 1 pagination test fails due to missing `historyPage`
10. 1 progress test fails due to type mismatch

### Test fixtures need update
11. `security.test.ts` imports `SESSION_TTL_MS` not exported from `memoryStore.ts`
12. `security.test.ts` passes `now` param to MemoryStore constructor (takes 0 args)
13. Device code `redeemDeviceCode` never returns `'expired'`/`'used'` states
    (token/route.ts checks for these)
