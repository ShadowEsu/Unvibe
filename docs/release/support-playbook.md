# Support playbook

## Triage

1. Acknowledge and assign severity: P0 privacy/security/cross-user/data-loss; P1 unusable core loop/deletion/signature; P2 degraded workflow; P3 cosmetic/suggestion.
2. Collect only build/checksum, macOS/editor version, timestamps, network/result class, and sanitized steps. Do not request repository contents, prompts, explanations, tokens, passwords, keys, or `.env` files.
3. For P0, pause invitations/distribution, preserve evidence, notify founder and security/privacy owner, and start the rollback plan. Do not promise breach scope or legal outcome.
4. Reproduce with synthetic code and disposable staging accounts. Link code fix, tests, release candidate, and verification evidence.
5. Close only after reporter confirmation or documented reproduction/fix evidence; record any deletion/privacy follow-up separately.

## Service targets for beta

- P0: acknowledge within 2 hours during the published support window.
- P1: same business day.
- P2/P3: two business days.

These are internal targets, not contractual SLAs. The founder must publish the actual support window and contact before inviting testers.
