# Private-beta tester onboarding

Before inviting a tester, provide the notarized artifact checksum, release notes, limitations, approved Terms/Privacy links, support address, cohort dates, and deletion instructions.

1. Confirm the tester uses a supported Apple-silicon Mac and VS Code or Cursor.
2. Verify the DMG checksum, install in Applications, and launch normally through Gatekeeper.
3. Explain why Accessibility permission is requested and how to revoke it in System Settings.
4. Sign in with the invited account. Never ask the tester for a password, token, repository, or provider key.
5. Start with a disposable or non-sensitive project. Review a small selection, change explanation level, ask a follow-up, use “Test me,” and confirm the event appears locally.
6. Try an offline review/restart and later sync. Do not deliberately paste real credentials to test filtering.
7. Submit feedback using the template. Redact code, file paths, repository names, email, screenshots, and request identifiers unless support explicitly needs a sanitized ID.
8. To leave beta, sign out, use in-app account deletion, uninstall, and optionally remove `~/Library/Application Support/Unvibe` after confirming needed exports/backups.

Testers should stop and contact support immediately for cross-account data, missing deletion, suspected secret transmission, data loss, repeated crashes, or Gatekeeper/signature warnings.
