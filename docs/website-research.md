# Unvibe launch-site research

Reviewed July 15, 2026. These references are used for product-storytelling principles only. No layouts, copy, illustrations, icons, or branded interaction sequences should be copied.

| Source | Observed pattern | Why it works | Unvibe interpretation |
| --- | --- | --- | --- |
| [Cursor](https://cursor.com/) | A direct category statement is followed immediately by a working product scene. | Visitors see the product doing its primary job before reading a feature inventory. | Put selected code and the Unvibe overlay in the hero, then let visitors drive the full loop. |
| [Linear](https://linear.app/) | Restrained navigation, precise copy, and large interface-led sections. | The page feels confident because each section has one argument. | Keep navigation compact and make every section answer one conversion question. |
| [Raycast](https://www.raycast.com/) | Native-utility framing and command-level product demonstrations. | It communicates speed and proximity to the user’s workflow. | Present Unvibe as a quiet desktop layer, with the editor remaining visually primary. |
| [Granola](https://www.granola.ai/) | A narrow use case is explained through recognizable moments rather than broad AI claims. | Specific situations create recognition and trust. | Use the moment after an agent completes a feature: the code works, but ownership is uncertain. |
| [Wispr Flow](https://wisprflow.ai/) | The surrounding tools change while the product layer remains consistent. | It makes a cross-application desktop product understandable at a glance. | Keep the Unvibe overlay constant and describe it as designed to work beside familiar tools, without implying partnerships. |
| [Arc](https://arc.net/) | Editorial pacing and strong product personality with limited visible choices. | A distinct point of view is easier to remember than a generic feature grid. | Use the line-to-project context ladder as Unvibe’s own signature storytelling device. |
| [Framer](https://www.framer.com/) | Product UI is the visual language; feature claims are paired with tangible outcomes. | The interface supplies proof instead of relying on decorative illustrations. | Prefer HTML product demonstrations and approved screenshots over generic artwork. |
| [Vercel](https://vercel.com/) | Progressive technical depth and disciplined hierarchy. | It serves fast-scanning visitors while allowing technical readers to go deeper. | Lead with the outcome, then reveal function, file, dependency, change, and architecture context. |
| [Superhuman](https://superhuman.com/) | Outcome-led headline, focused calls to action, and polished interaction details. | The page sells a changed working experience, not a list of controls. | Frame the benefit as maintaining and owning AI-written code tomorrow. |
| [Notion](https://www.notion.com/) | Broad capability is made legible through grouped workflows and recognizable product views. | Visitors can map the product onto their own work. | Show overlay, project, study, progress, and notebook as one connected learning loop. |
| [Attio](https://attio.com/) | Editorial section composition varies instead of repeating three-card grids. | Visual rhythm makes a long product story feel intentional. | Alternate full-width demonstrations, split editorial layouts, horizontal screenshots, and compact trust sections. |
| [Resend](https://resend.com/) | Developer-facing copy is concise and the product proof uses realistic technical material. | Credibility comes from specificity and readable code. | Keep examples plausible, code legible, and claims limited to current implementation. |

## Principles adopted

1. Communicate one job in the hero: understand AI-written code without leaving the workflow.
2. Demonstrate before describing.
3. Turn the central user tension into a recognizable moment, without shaming AI-assisted development.
4. Keep one primary conversion action: join the private beta.
5. Use one signature visual idea repeatedly enough to become memorable.
6. Keep product status and privacy language concrete and verifiable.
7. Recompose complex demonstrations for mobile instead of shrinking desktop windows.
8. Treat reduced motion, pause, replay, and keyboard control as product quality, not extras.

## Patterns intentionally rejected

- Invented testimonials, customer logos, user counts, download counts, or security certifications.
- Neon AI glows, floating-card wallpaper, uncontrolled node graphs, and decorative WebGL.
- Repeated feature-card grids.
- Slow character-by-character typing and autoplay video without controls.
- Partnership language around Cursor, VS Code, GitHub, or other tools.
- Download-first messaging while access is still invitation-based.
